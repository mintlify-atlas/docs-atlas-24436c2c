---
title: Environment Variables
description: Complete reference of all environment variables for configuring Zou
---

Zou uses environment variables for configuration. All configuration is defined in `zou/app/config.py`.

## Core Settings

<ParamField path="DEBUG" type="boolean" default="False">
  Enable debug mode. Shows detailed error messages and enables auto-reload.
  
  <Warning>Never enable in production environments</Warning>
</ParamField>

<ParamField path="DEBUG_HOST" type="string" default="127.0.0.1">
  Host address for debug server.
</ParamField>

<ParamField path="DEBUG_PORT" type="integer" default="5000">
  Port for debug server.
</ParamField>

<ParamField path="SECRET_KEY" type="string" default="mysecretkey">
  Secret key for session encryption and JWT tokens.
  
  <Warning>Must be changed in production. Use a strong random string</Warning>
</ParamField>

## Database Configuration

<ParamField path="DB_DRIVER" type="string" default="postgresql+psycopg">
  SQLAlchemy database driver. Uses psycopg (version 3) by default.
</ParamField>

<ParamField path="DB_HOST" type="string" default="localhost">
  PostgreSQL database host address.
</ParamField>

<ParamField path="DB_PORT" type="string" default="5432">
  PostgreSQL database port.
</ParamField>

<ParamField path="DB_USERNAME" type="string" default="postgres">
  Database username for authentication.
</ParamField>

<ParamField path="DB_PASSWORD" type="string" default="mysecretpassword">
  Database password for authentication.
  
  <Warning>Change this in production environments</Warning>
</ParamField>

<ParamField path="DB_DATABASE" type="string" default="zoudb">
  Name of the PostgreSQL database.
</ParamField>

### Connection Pool Settings

<ParamField path="DB_POOL_SIZE" type="integer" default="30">
  Number of database connections to maintain in the pool.
</ParamField>

<ParamField path="DB_MAX_OVERFLOW" type="integer" default="60">
  Maximum number of connections that can be created beyond pool_size.
</ParamField>

<ParamField path="DB_POOL_PRE_PING" type="boolean" default="True">
  Verify connections before using them. Prevents using stale connections.
</ParamField>

<ParamField path="DB_POOL_RECYCLE" type="integer" default="3600">
  Recycle connections after this many seconds (prevents long-lived connections).
</ParamField>

<ParamField path="DB_POOL_RESET_ON_RETURN" type="string" default="commit">
  Reset connections when returning to pool. Options: `commit`, `rollback`, `none`.
  
  Cleans up transaction state to prevent issues between requests.
</ParamField>

## Authentication & Security

<ParamField path="AUTH_STRATEGY" type="string" default="auth_local_classic">
  Authentication strategy. Options:
  - `auth_local_classic` - Standard username/password authentication
  - LDAP authentication (requires additional LDAP configuration)
  - SAML authentication (requires additional SAML configuration)
</ParamField>

<ParamField path="BCRYPT_LOG_ROUNDS" type="integer" default="12">
  Bcrypt password hashing rounds. Higher values increase security but slow down authentication.
</ParamField>

<ParamField path="MIN_PASSWORD_LENGTH" type="integer" default="8">
  Minimum password length requirement.
</ParamField>

<ParamField path="ENFORCE_2FA" type="boolean" default="False">
  Require two-factor authentication for all users.
</ParamField>

<ParamField path="2FA_EXEMPT_USERS" type="string" default="">
  Comma-separated list of user emails exempt from mandatory 2FA requirement.
  
  Example: `user1@example.com,user2@example.com`
</ParamField>

<ParamField path="PROTECTED_ACCOUNTS" type="string" default="">
  Semicolon-separated list of protected account emails that cannot be modified.
</ParamField>

<ParamField path="USER_LIMIT" type="integer" default="100">
  Maximum number of users allowed in the system.
</ParamField>

### JWT Token Configuration

JWT tokens are managed automatically. The following are configured in code:

- **Access token expires**: 7 days
- **Refresh token expires**: 15 days
- **Token locations**: cookies and headers
- **Cookie SameSite**: Lax

<ParamField path="CLIENT_CACHE_MAX_AGE" type="integer" default="604800">
  Client-side cache max age in seconds (default: 7 days).
</ParamField>

## Example Configuration

```bash
# Core
export SECRET_KEY="your-secret-key-here"
export DEBUG=False

# Database
export DB_HOST="localhost"
export DB_PORT="5432"
export DB_USERNAME="postgres"
export DB_PASSWORD="secure-password"
export DB_DATABASE="zoudb"
export DB_POOL_SIZE="30"

# Redis
export KV_HOST="localhost"
export KV_PORT="6379"

# Storage
export PREVIEW_FOLDER="/var/zou/previews"
export FS_BACKEND="local"

# Mail
export MAIL_ENABLED=True
export MAIL_SERVER="smtp.example.com"
export MAIL_PORT="587"
export MAIL_USE_TLS=True
export MAIL_USERNAME="notifications@studio.com"
export MAIL_PASSWORD="mail-password"
export MAIL_DEFAULT_SENDER="noreply@studio.com"

# Domain
export DOMAIN_NAME="kitsu.studio.com"
export DOMAIN_PROTOCOL="https"

# Event Stream
export EVENT_STREAM_HOST="0.0.0.0"
export EVENT_STREAM_PORT="5001"
```
