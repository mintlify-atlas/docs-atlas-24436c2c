## Core Project Endpoints

### Get Open Projects

<api-endpoint verb="GET" path="/api/data/projects/open" />

Retrieve all currently active projects. This endpoint returns projects with "Open", "open", or "Active" status, along with enriched data including metadata descriptors, task types, and task statuses.

**Authentication**: Required (JWT)

**Query Parameters**

<ParamField query="name" type="string">
  Filter projects by name
</ParamField>

**Response**

<ResponseField name="projects" type="array">
  <ResponseField name="id" type="string">
    Project unique identifier
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Project name
  </ResponseField>
  
  <ResponseField name="code" type="string">
    Project code
  </ResponseField>
  
  <ResponseField name="description" type="string">
    Project description
  </ResponseField>
  
  <ResponseField name="project_status_id" type="string">
    Project status identifier
  </ResponseField>
  
  <ResponseField name="production_type" type="string">
    Type of production (short, tvshow, featurefilm, etc.)
  </ResponseField>
  
  <ResponseField name="production_style" type="string">
    Visual style (2d, 3d, vfx, etc.)
  </ResponseField>
  
  <ResponseField name="fps" type="string">
    Frames per second
  </ResponseField>
  
  <ResponseField name="ratio" type="string">
    Aspect ratio
  </ResponseField>
  
  <ResponseField name="resolution" type="string">
    Video resolution
  </ResponseField>
  
  <ResponseField name="start_date" type="string">
    Production start date
  </ResponseField>
  
  <ResponseField name="end_date" type="string">
    Production end date
  </ResponseField>
  
  <ResponseField name="descriptors" type="array">
    Array of metadata descriptors for this project
  </ResponseField>
  
  <ResponseField name="task_types_priority" type="object">
    Task type IDs mapped to their priority values
  </ResponseField>
  
  <ResponseField name="task_statuses_link" type="object">
    Task status IDs mapped to priority and roles_for_board
  </ResponseField>
  
  <ResponseField name="first_episode_id" type="string">
    First episode ID (for TV shows)
  </ResponseField>
</ResponseField>

**Example Request**

```bash
curl -X GET "https://kitsu.example.com/api/data/projects/open" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Create Metadata Descriptor

<api-endpoint verb="POST" path="/api/data/projects/{project_id}/metadata-descriptors" />

Create a new metadata descriptor (custom field) for the project.

**Authentication**: Required (JWT)

**Permissions**: Manager on project

**Path Parameters**

<ParamField path="project_id" type="string" required>
  Project unique identifier
</ParamField>

**Request Body**

<ParamField body="name" type="string" required>
  Display name for the field
</ParamField>

<ParamField body="entity_type" type="string">
  Entity type: "Asset", "Shot", "Edit", "Episode", or "Sequence" (default: "Asset")
</ParamField>

<ParamField body="data_type" type="string" required>
  Data type: "string", "number", "boolean", "list", "checklist", "tags", etc.
</ParamField>

<ParamField body="choices" type="array">
  List of choices for list/dropdown fields
</ParamField>

<ParamField body="for_client" type="string">
  Whether visible to clients: "True" or "False" (default: "False")
</ParamField>

<ParamField body="departments" type="array">
  Array of department IDs that can access this field
</ParamField>

**Response**

<ResponseField name="descriptor" type="object">
  Created metadata descriptor object
</ResponseField>

**Error Codes**

- **400**: Invalid parameters (wrong entity_type, data_type, or empty name)

**Example Request**

```bash
curl -X POST "https://kitsu.example.com/api/data/projects/a24a6ea4-ce75-4665-a070-57453082c25/metadata-descriptors" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Difficulty",
    "entity_type": "Asset",
    "data_type": "list",
    "choices": ["easy", "medium", "hard"],
    "for_client": "False",
    "departments": []
  }'
```

**Example Response**

```json
{
```
