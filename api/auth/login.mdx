## Login

<RequestExample>

```bash cURL
curl -X POST https://zou.example.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "mysecretpassword"
  }'
```

```python Python
import requests

response = requests.post(
    "https://zou.example.com/api/auth/login",
    json={
        "email": "admin@example.com",
        "password": "mysecretpassword"
    }
)

data = response.json()
access_token = data["access_token"]
```

```javascript JavaScript
const response = await fetch('https://zou.example.com/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'admin@example.com',
    password: 'mysecretpassword'
  })
});

const data = await response.json();
const accessToken = data.access_token;
```

</RequestExample>

### POST /api/auth/login

Log in user by creating and registering JWT auth tokens. Login is based on email and password. If no user matches the given email, it fallbacks to a desktop ID. This is useful for desktop tools that don't know the user email.

The endpoint also supports two-factor authentication via TOTP, Email OTP, FIDO, and recovery codes.

#### Request Body

<ParamField body="email" type="string" required>
  User email address
  
  Example: `admin@example.com`
</ParamField>

<ParamField body="password" type="string" required>
  User password
  
  Example: `mysecretpassword`
</ParamField>

<ParamField body="totp" type="string">
  TOTP verification code for two-factor authentication
  
  Example: `123456`
</ParamField>

<ParamField body="email_otp" type="string">
  Email OTP verification code for two-factor authentication
  
  Example: `123456`
</ParamField>

<ParamField body="fido_authentication_response" type="object">
  FIDO authentication response for WebAuthn
</ParamField>

<ParamField body="recovery_code" type="string">
  Recovery code for two-factor authentication
  
  Example: `ABCD-EFGH-IJKL-MNOP`
</ParamField>

#### Response

<ResponseField name="login" type="boolean">
  Whether the login was successful
</ResponseField>

<ResponseField name="user" type="object">
  User information including id, email, first_name, last_name, role, and other profile data
</ResponseField>

<ResponseField name="organisation" type="object">
  Organisation information
</ResponseField>

<ResponseField name="access_token" type="string">
  JWT access token for API authentication
</ResponseField>

<ResponseField name="refresh_token" type="string">
  JWT refresh token to obtain new access tokens
</ResponseField>

<ResponseField name="two_factor_authentication_required" type="boolean">
  Present when 2FA setup is enforced but not yet configured
</ResponseField>

#### Error Responses

<ResponseField name="login" type="boolean" default="false">
  Indicates login failure
</ResponseField>

<ResponseField name="error" type="boolean">
  Indicates an error occurred
</ResponseField>

<ResponseField name="default_password" type="boolean">
  User must change their default password
</ResponseField>

<ResponseField name="token" type="string">
  Password reset token when default_password is true
</ResponseField>

<ResponseField name="missing_OTP" type="boolean">
  Two-factor authentication code is required but not provided
</ResponseField>

<ResponseField name="wrong_OTP" type="boolean">
  Provided OTP code is incorrect
</ResponseField>

<ResponseField name="too_many_failed_login_attemps" type="boolean">
  Account locked due to too many failed login attempts
</ResponseField>

<ResponseField name="message" type="string">
  Error message description
</ResponseField>

#### Status Codes

- `200` - Login successful
- `400` - Login failed (wrong credentials, missing OTP, or validation error)
- `401` - User is inactive
- `409` - Authentication strategy not properly configured
- `500` - Server error (database unreachable or internal error)

## Check Authentication Status

<RequestExample>

```bash cURL
curl -X GET https://zou.example.com/api/auth/authenticated \
  -H "Authorization: Bearer <access_token>"
```

```python Python
import requests

response = requests.get(
    "https://zou.example.com/api/auth/authenticated",
    headers={"Authorization": f"Bearer {access_token}"}
)

data = response.json()
```

```javascript JavaScript
const response = await fetch('https://zou.example.com/api/auth/authenticated', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});

const data = await response.json();
```

</RequestExample>

### GET /api/auth/authenticated

Check if the user is still authenticated. Returns user information if authenticated. This endpoint can be used by third-party tools, especially browser frontends, to verify if the current user is still logged in.

#### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Format: `Bearer <access_token>`
</ParamField>

#### Response

<ResponseField name="authenticated" type="boolean">
  Always true when request succeeds
</ResponseField>

<ResponseField name="user" type="object">
  Current user information including profile data, role, and FIDO devices
</ResponseField>

<ResponseField name="organisation" type="object">
  Organisation information (sensitive data included for admin users)
</ResponseField>

#### Status Codes

- `200` - User authenticated
- `401` - Person not found or invalid token

### GET /api/auth/logout

Log user out by revoking auth tokens. Once logged out, the current user cannot access the API anymore.

#### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Format: `Bearer <access_token>`
</ParamField>

#### Response

<ResponseField name="logout" type="boolean">

</ResponseField>

#### Status Codes

- `200` - Logout successful
- `401` - Invalid or missing token
- `500` - Access token not found

### POST /api/auth/register

Allow a user to register themselves to the service.

#### Request Body

  User's last name
</ParamField>

  Whether registration was successful
</ResponseField>

  Indicates an error occurred
</ResponseField>

<ResponseField name="message" type="string">
  Error message when registration fails
</ResponseField>

#### Status Codes

- `201` - Registration successful
- `400` - Invalid password, email, or passwords don't match

## Change Password

<RequestExample>

```bash cURL
curl -X POST https://zou.example.com/api/auth/change-password \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "old_password": "currentpassword",
    "password": "newpassword123",
    "password_2": "newpassword123"
  }'
```

```python Python
import requests

response = requests.post(
    "https://zou.example.com/api/auth/change-password",
    headers={"Authorization": f"Bearer {access_token}"},
    json={
        "old_password": "currentpassword",
        "password": "newpassword123",
        "password_2": "newpassword123"
    }
)
```

```javascript JavaScript
const response = await fetch('https://zou.example.com/api/auth/change-password', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    old_password: 'currentpassword',
    password: 'newpassword123',
    password_2: 'newpassword123'
  })
});
```

</RequestExample>

### POST /api/auth/change-password

Allow the user to change their password. Requires current password for verification and password confirmation to ensure accuracy.

#### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Format: `Bearer <access_token>`
</ParamField>

#### Request Body

<ParamField body="old_password" type="string" required>
  Current password
</ParamField>

<ParamField body="password" type="string" required>
  New password
</ParamField>

<ParamField body="password_2" type="string" required>
  New password confirmation
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Whether password change was successful
</ResponseField>

<ResponseField name="error" type="boolean">
  Indicates an error occurred
</ResponseField>

<ResponseField name="message" type="string">
  Error message description
</ResponseField>

#### Status Codes

- `200` - Password changed successfully
- `400` - Invalid password, passwords don't match, old password is wrong, or user is inactive

<Note>
  After a successful password change, a confirmation email is sent to the user's email address.
</Note>

## Password Reset

### Request Password Reset

<RequestExample>

```bash cURL
curl -X POST https://zou.example.com/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com"
  }'
```

```python Python
import requests

response = requests.post(
    "https://zou.example.com/api/auth/reset-password",
    json={"email": "user@example.com"}
)
```

```javascript JavaScript
const response = await fetch('https://zou.example.com/api/auth/reset-password', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com'
  })
});
```

</RequestExample>

### POST /api/auth/reset-password

Send a password reset token by email to the user. It uses a classic scheme where a token is sent by email with a 2-hour expiration.

#### Request Body

<ParamField body="email" type="string" required>
  User email address
  
  Example: `user@example.com`
</ParamField>

#### Response

<ResponseField name="success" type="string">
  Message indicating reset token was sent
</ResponseField>

<ResponseField name="error" type="boolean">
  Indicates an error occurred
</ResponseField>

<ResponseField name="message" type="string">
  Error message description
</ResponseField>

#### Status Codes

- `200` - Reset token sent successfully
- `400` - Email not listed in database or user is inactive

### Reset Password with Token

<RequestExample>

```bash cURL
curl -X PUT https://zou.example.com/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "token": "reset-token-from-email",
    "password": "newpassword123",
    "password2": "newpassword123"
  }'
```

```python Python
import requests

response = requests.put(
    "https://zou.example.com/api/auth/reset-password",
    json={
        "email": "user@example.com",
        "token": "reset-token-from-email",
        "password": "newpassword123",
        "password2": "newpassword123"
    }
)
```

```javascript JavaScript
const response = await fetch('https://zou.example.com/api/auth/reset-password', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com',
    token: 'reset-token-from-email',
    password: 'newpassword123',
    password2: 'newpassword123'
  })
});
```

</RequestExample>

### PUT /api/auth/reset-password

Allow a user to change their password when they forget it. It uses a token sent by email to verify it is the user who requested the password reset.

#### Request Body

<ParamField body="email" type="string" required>
  User email address
  
  Example: `user@example.com`
</ParamField>

<ParamField body="token" type="string" required>
  Password reset token received via email
</ParamField>

<ParamField body="password" type="string" required>
  New password
</ParamField>

<ParamField body="password2" type="string" required>
  New password confirmation
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Whether password reset was successful
</ResponseField>

<ResponseField name="error" type="boolean">
  Indicates an error occurred
</ResponseField>

<ResponseField name="message" type="string">
  Error message description
</ResponseField>

#### Status Codes

- `200` - Password reset successful
- `400` - Invalid password, passwords don't match, wrong or expired token, or inactive user

<Warning>
  Reset tokens expire after 2 hours and can only be used once.
</Warning>
